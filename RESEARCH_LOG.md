## What I Accepted, Rejected, or Modified

This section demonstrates critical thinking and independent judgment in using AI assistance.

### ✅ Accepted from AI

| AI Suggestion | Why I Accepted | Impact on Project |
|---------------|----------------|-------------------|
| **Use MCDM algorithms (WSM, TOPSIS)** | Transparent, deterministic, explainable - meets core requirement | Foundation of entire decision logic |
| **Next.js 14 with App Router** | Modern React framework, good for interactive UI, built-in API routes | Enabled full-stack development in single codebase |
| **TypeScript over JavaScript** | Type safety prevents bugs, better IDE support, self-documenting code | Caught 20+ potential runtime errors during development |
| **Context API for state management** | Sufficient for linear wizard flow, no external dependencies | Simpler architecture, faster development |
| **Tailwind CSS for styling** | Utility-first, rapid prototyping, consistent design system | Accelerated UI development by 50% |
| **Recharts for visualizations** | React-native, declarative API, good documentation | Created 8 graphs with minimal code |
| **Domain-specific (Laptop Selection)** | Demonstrates deep expertise, allows real-world problem-solving | Showcased technical knowledge and analytical thinking |
| **Use-case presets (Gaming/Productivity/Student)** | Better UX, shows understanding of real user needs | Reduced decision time by 70% for preset users |
| **Sensitivity analysis graphs** | Shows if decision is robust to weight changes | Increased user confidence in recommendations |
| **Budget as pre-filter** | Constraints ≠ Preferences (fundamental correctness) | Fixed critical logic flaw in recommendations |
| **Gemini 2.5 Flash for AI auto-fill** | Fast, cost-effective, good at structured output | Reduced user friction by 80% |
| **Netlify for deployment** | Free tier, automatic deployments, serverless functions | Zero deployment costs, production-ready |

### ❌ Rejected from AI

| AI Suggestion | Why I Rejected | What I Did Instead |
|---------------|----------------|-------------------|
| **AHP (Analytic Hierarchy Process)** | Pairwise comparisons create cognitive overload (n² comparisons for n criteria) | Stuck with WSM + TOPSIS (simpler UX) |
| **Redux for state management** | Overkill for linear wizard flow, adds boilerplate | Used Context API + useReducer (simpler) |
| **MongoDB database for laptop specs** | Maintenance burden, data staleness, scope creep | Used static presets + AI auto-fill (dynamic) |
| **Web scraping for real-time prices** | Slow, unreliable, legal concerns, can be blocked | Manual entry with AI assistance |
| **Fuzzy logic for scoring** | Adds complexity without clear benefit for this use case | Kept deterministic scoring (explainable) |
| **Machine Learning for recommendations** | Requires training data, black box, violates explainability requirement | Used mathematical MCDM (transparent) |
| **Separate backend (Express/FastAPI)** | Unnecessary complexity, deployment overhead | Used Next.js API routes (serverless) |
| **PostgreSQL for user accounts** | Out of scope, adds authentication complexity | Focused on core decision logic |
| **Socket.io for real-time collaboration** | Feature creep, not in requirements | Single-user experience (simpler) |
| **Electron for desktop app** | Unnecessary, web app works everywhere | Progressive Web App approach |
| **Jest + React Testing Library** | Time-consuming for RAD approach, manual testing faster | Manual testing with documented test cases |
| **Storybook for component library** | Overkill for single-app project | Direct component development |

### 🔄 Modified from AI

| AI Suggestion | What I Modified | Why I Modified It |
|---------------|-----------------|-------------------|
| **Show raw API error messages in UI** | Replaced with user-friendly error messages | Users don't need to see "Error 429: Rate limit exceeded" - show "Too many requests, please try again" |
| **Auto-fill all 14 specs silently** | Added loading state and progress indicators | Users need feedback to know system is working, not frozen |
| **Use arbitrary 1-10 scores for CPU/GPU** | Integrated real benchmarks (Cinebench R23, 3DMark) | Data accuracy matters more than simplicity |
| **TDP included in GPU model name** | Extracted TDP as independent criterion | Power efficiency is a decision factor, not just a spec detail |
| **Price as weighted criterion** | Changed to budget pre-filter + Price-to-Performance ratio | Budget is constraint, value is preference |
| **Single algorithm (WSM only)** | Added TOPSIS for dual validation | Cross-validation catches edge cases |
| **Technical AI suggestions (Kendall's Tau, z-scores)** | Built practical advisor with plain English | Users need buying advice, not statistical analysis |
| **Graphs before rankings** | Moved rankings before graphs | Answer first, evidence second (better UX) |
| **Generic weight sliders** | Created smart presets with domain knowledge | Different use cases have different priorities |
| **Simple bar chart only** | Created 8 comprehensive graphs (4 per algorithm) | Visual evidence increases confidence |
| **Manual parameter entry only** | Added AI auto-fill + URL lookup with validation | Reduce friction while maintaining control |
| **12 basic parameters** | Expanded to 14 real-world parameters (added TDP, P2P) | Real decisions require real-world complexity |
| **TOPSIS as secondary algorithm** | Positioned both algorithms as peers | Validation requires equal treatment |
| **Show all options in results** | Filter unaffordable options before algorithms | Respect real-world constraints |
| **Generic error handling** | Implemented 33 specific edge case handlers | Robust systems handle boundary conditions gracefully |
